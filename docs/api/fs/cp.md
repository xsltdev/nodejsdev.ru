# cp

## Promise

!!!warning "Стабильность: 1 - Экспериментальный"

    На эту функцию не распространяются семантические правила управления версиями. Изменения или удаление, не совместимые с предыдущими версиями, могут произойти в любом будущем выпуске. Использование этой функции не рекомендуется в производственных средах.

```js
fsPromises.cp(src, dest[, options])
```

???quote "История"

    | Version | Changes                                                                                                   |
    | ------- | --------------------------------------------------------------------------------------------------------- |
    | v17.6.0 | Accepts an additional verbatimSymlinks option to specify whether to perform path resolution for symlinks. |
    | v16.7.0 | Added in: v16.7.0                                                                                         |

**Параметры:**

-   `src` : `<string>` | `<URL>` &mdash; исходный путь для копирования
-   `dest` : `<string>` | `<URL>` &mdash; путь назначения для копирования
-   `options` : `<Object>`
    -   `dereference` = `false` : `<boolean>` &mdash; символические ссылки разыменования
    -   `errorOnExist` = `false` : `<boolean>` &mdash; когда `force` ложно, а пункт назначения существует, выдать ошибку
    -   `filter` = `undefined` : `<Function>` &mdash; функция для фильтрации скопированных файлов/каталогов. Верните `true`, чтобы скопировать элемент, и `false`, чтобы игнорировать его. Также может возвращать `Promise`, которое принимает значение `true` или `false`
        -   `src` : `<string>` &mdash; исходный путь для копирования
        -   `dest` : `<string>` &mdash; путь назначения для копирования
        -   Возвращает: `<boolean>` | `<Promise>`
    -   `force` = `true` : `<boolean>` &mdash; перезаписать существующий файл или каталог. Операция копирования будет игнорировать ошибки, если вы установите для этого параметра значение `false` и место назначения существует. Используйте параметр `errorOnExist`, чтобы изменить это поведение
    -   `preserveTimestamps` = `false` : `<boolean>` &mdash; при значении `true` разрешение пути для символических ссылок будет пропущено
    -   `recursive` = `false` : `<boolean>` &mdash; копировать каталоги рекурсивно
    -   `verbatimSymlinks` = `false` : `<boolean>` &mdash; при значении `true` разрешение пути для символических ссылок будет пропущено

**Возвращает:**

-   `<Promise>` &mdash; Выполняется с `undefined` в случае успеха.

Асинхронно копирует всю структуру каталогов из `src` в `dest`, включая подкаталоги и файлы.

При копировании каталога в другой каталог подстановочные знаки не поддерживаются, и поведение аналогично `cp dir1/ dir2/`.
